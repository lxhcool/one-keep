# 应用签名配置指南

## 已生成的文件

- `android/app/release.keystore` — Release 签名密钥库
- `android/key.properties` — 签名配置文件（已被 `.gitignore` 排除）

## 生成方法（如需重新生成）

```bash
# 确保在 client 目录下
cd client

# 生成 release keystore（请替换 YOUR_NAME/YOUR_UNIT 等信息）
keytool -genkey -v \
  -keystore android/app/release.keystore \
  -alias onekeep \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -storepass your_store_password \
  -keypass your_key_password \
  -dname "CN=厘清, OU=Development, O=LXH, L=Shanghai, ST=Shanghai, C=CN"
```

## 密钥信息

| 项目 | 值 |
|------|-----|
| 密钥库路径 | `android/app/release.keystore` |
| 别名 | `onekeep` |
| 算法 | RSA 2048 |
| 有效期 | 10000 天 |

> 请务必备份 `release.keystore` 文件！丢失后将无法更新已上架的应用。

## 构建 Release 版本

```bash
cd client
flutter build apk --release
```

APK 输出路径：`build/app/outputs/flutter-apk/app-release.apk`
