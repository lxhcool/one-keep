# 后台端 DDD 架构设计

聚焦后台管理端职责，确保运营能力、权限治理和审计链路完整。

## 工作流

### 1. 识别后台业务场景

- 提取后台用户角色与职责（运营、审核、客服、管理员）。
- 界定后台与客户端/服务端的边界。
- 标记高风险操作与合规要求。

### 2. 划分后台模块边界

- 定义后台功能域与子模块（配置、查询、审批、运营操作）。
- 设计页面到后台用例映射。
- 约束跨模块调用与数据访问边界。

### 3. 设计权限与审计模型

- 定义角色、权限点、资源范围与数据权限。
- 定义关键操作审计字段与追踪链路。
- 定义危险操作确认、回滚与双人复核策略。

### 4. 设计后台契约与治理

- 定义后台 API 契约与查询模型。
- 设计分页、筛选、导出、批量操作约束。
- 设计异常反馈、告警与操作留痕策略。

### 5. 输出架构文档

- 使用 `references/admin-architecture-template.md`。
- 输出需可直接指导后台模块实现与权限评审。

## 质量标准

- 后台职责边界清晰，不侵入核心领域规则。
- 权限与审计闭环，可追责可回溯。
- 高风险操作具备防误用与补救机制。
