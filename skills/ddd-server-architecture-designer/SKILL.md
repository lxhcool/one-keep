# 服务端 DDD 架构设计

聚焦服务端领域模型与契约治理，确保业务规则在后端被正确承载。

## 工作流

### 1. 澄清服务端领域边界

- 从 PRD 与交互中提取核心域与支撑域。
- 划分限界上下文与上下游关系。
- 记录关键术语，避免跨上下文歧义。

### 2. 建模聚合与不变式

- 定义实体、值对象、聚合、领域服务、领域事件。
- 明确命令/查询职责边界。
- 明确事务边界与并发一致性要求。

### 3. 设计应用服务与集成层

- 定义应用服务、仓储接口、外部系统适配器。
- 定义 API 契约、事件契约、幂等策略。
- 设计失败重试、补偿、降级策略。

### 4. 设计持久化与治理

- 规划数据模型与迁移边界。
- 规划认证授权、审计、可观测性要点。
- 标注性能瓶颈与容量风险。

### 5. 输出架构文档

- 使用 `references/server-architecture-template.md`。
- 输出需可直接指导服务端实现与评审。

## 质量标准

- 聚合边界与业务不变式一致。
- 契约明确、可测试、可版本化。
- 一致性与补偿策略可落地执行。
