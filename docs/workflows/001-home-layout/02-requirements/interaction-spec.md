# 交互规格 — OneKeep 财务App 首页

> 本文档是 [prd.md](prd.md) 的交互补充，面向 UI 开发者和测试工程师，描述每个可交互元素的状态机、动效规格和边界行为。

---

## 1. 页面整体结构与分层

```
Stack (全屏 402×874)
├── Layer 0: 内容层（可滚动）
│   ├── StatusBar（系统级，不可滚动）
│   └── ScrollView（ContentArea，含顶部栏/结余卡片/收支统计/近期账单）
├── Layer 1: TabBar（底部固定，不随滚动移动）
└── Layer 2: FAB（悬浮固定，叠于 TabBar 上方）
```

## 2. 顶部栏（TopBar）

### 2.1 用户信息区（UserInfo）

| 元素 | 状态 | 展示逻辑 |
| --- | --- | --- |
| 问候语 | 静态文本 | 06:00–11:59 → "早上好，"；12:00–17:59 → "下午好，"；18:00–05:59 → "晚上好，" |
| 用户名 | 静态文本 | 从用户 Profile 读取，超 20 字符末尾截断加省略号 |
| 头像 | 静态图标 | v1.0 使用渐变底色 + user 图标占位；v2.0 支持真实头像图片 |

### 2.2 搜索图标

| 状态 | 视觉 | 行为 |
| --- | --- | --- |
| Default | `$text-tertiary` 灰色图标 | — |
| Pressed | 轻微透明度降低（opacity 0.6） | v1.0：弹出 Toast "搜索功能即将上线"，3s 自动消失 |

### 2.3 通知图标

| 状态 | 视觉 | 行为 |
| --- | --- | --- |
| Default | `$text-primary` 深色图标 | — |
| Pressed | 透明度降低 | v1.0：弹出 Toast "通知功能即将上线"，3s 自动消失 |
| 有未读通知 | 右上角红点（v2.0） | 不在 v1.0 范围内 |

## 3. 本月结余卡片（BalanceCard）

### 3.1 状态机

```
[进入首页]
    ↓
[Loading] — 显示 Shimmer 骨架屏（卡片区域等比灰块动画）
    ↓ 数据返回成功                    ↓ 数据返回失败 / 超时
[Success] — 渲染真实金额            [Error] — 显示"数据加载失败 ↺"
    ↓ 用户下拉刷新                    ↓ 点击重试
[Loading] ← ─────────────────────── [Loading]
```

### 3.2 Success 态字段规则

| 字段 | 格式 | 特殊情况 |
| --- | --- | --- |
| 结余金额 | `¥#,###.##`（千分位分隔，两位小数） | 负数前加负号：`¥-1,230.00` |
| 环比图标 | 环比 ≥ 0 → `trending-up`；< 0 → `trending-down` | 数据缺失时整行隐藏 |
| 环比文本 | `+XX%` / `-XX%`（含符号） | 数据缺失时整行隐藏 |

### 3.3 错误态

- 卡片保持绿色渐变背景
- 中心显示灰白色文字："数据加载失败" + 重试图标
- 点击整个卡片区域触发重试

---

## 4. 收支统计（IncomeExpenseRow）

### 4.1 状态机

与 BalanceCard 共享同一加载流程，Loading / Success / Error 三态。

### 4.2 Success 态

- 收入金额：`¥#,###`（整数，无小数）
- 支出金额：同上，展示为正数（设计稿未显示负号，以正数展示支出总额）
- 两卡片等宽（`fill_container` 均分）

### 4.3 Error 态

- 两张卡片内金额位置均显示 `--`
- 不单独显示重试按钮（随页面级下拉刷新恢复）

## 5. 近期账单（RecentTransactions）

### 5.1 状态机

```
[进入首页]
    ↓
[Loading] — Shimmer 列表项（3 条占位）
    ↓ 有数据          ↓ 无数据          ↓ 失败
[Success]          [Empty]           [Error]
```

### 5.2 Success 态列表渲染规则

- 取最新数据，按日期降序分组，最多展示 2 个日期分组，共 ≤ 5 条
- 日期标签 `DateLabel`：今日显示"今天"，昨日显示"昨天"，更早显示 `MM月DD日`
- 账单项（`TransactionItem`）字段：

| 字段 | 规则 |
| --- | --- |
| 分类图标 | 按 category 枚举匹配图标和底色，未知分类用默认图标 |
| 标题（title） | 最多 1 行，超长省略 |
| 副标题（subtitle） | 固定为"收入"或"支出"文字 |
| 金额 | 支出：`-#,###.##`，`$text-primary` 色；收入：`+#,###.##`，`$accent-green` 色 |

### 5.3 Empty 态

- 隐藏列表区域
- 展示空状态插图（居中）
- 文案："暂无账单，点击 + 记一笔"
- 文案中的 "+" 可点击，触发 FAB 相同行为（弹出记账弹窗）

### 5.4 Error 态

- 显示 "加载失败" 文字 + "重试" 按钮
- 点击"重试"重新请求账单数据

### 5.5 "查看全部" 按钮

| 状态 | 行为 |
| --- | --- |
| Default | 灰色文字 + chevron-right 图标 |
| Pressed | 轻微透明度降低 |
| 点击 | 切换至 TabBar "账单" tab，导航至账单页 |

### 5.6 账单列表项点击

| v | 行为 |
| --- | --- |
| v1.0 | 暂不响应（或 Toast 提示"详情功能即将上线"） |
| v2.0 | 跳转至账单详情页 |

## 6. FAB 浮动按钮

### 6.1 状态机

| 状态 | 视觉 | 触发 |
| --- | --- | --- |
| Default | 绿色渐变圆形，白色 "+" 图标，绿色投影 | — |
| Pressed | 略微缩小（scale 0.95） + 投影减弱 | 手指按下 |
| Released | 恢复 Default | 手指抬起 |
| Loading（记账中） | "+" 变为 loading spinner（可选） | 提交记账时 |

### 6.2 位置规则

- 水平居中，Y 轴固定于 TabBar 上边缘向上偏移 24px
- 使用 `Stack` + `Positioned` 实现，不随页面内容滚动

### 6.3 点击行为

1. 触发弹出快速记账弹窗（底部向上滑入，overlay 遮罩）
2. 记账弹窗关闭时：
   - 若用户保存了新账单 → 首页账单列表静默刷新（无骨架屏，直接更新数据）
   - 若用户取消 → 无任何变化

## 7. 底部 TabBar

### 7.1 Tab 状态

| Tab | Active 视觉 | Inactive 视觉 |
| --- | --- | --- |
| 首页（house） | 图标 + 文字均为 `$accent-green` | 图标 + 文字均为 `$text-tertiary` |
| 统计（chart-bar） | 同上 | 同上 |
| 账单（receipt） | 同上 | 同上 |
| 我的（user） | 同上 | 同上 |

### 7.2 切换行为

- 点击非当前 Tab：切换页面，图标/文字颜色即时更新
- 点击当前 Tab：滚动至页面顶部（如当前页已滚动）
- 切换动效：页面切换使用 Fade 或 Slide 过渡（具体由 App 全局路由配置决定）

### 7.3 安全区

- TabBar 底部 padding = 34px（对应 iPhone 底部 Home Indicator 区域）
- Android 无 Home Indicator 时 padding 可减小（通过 `MediaQuery.padding.bottom` 动态计算）

## 8. 页面级下拉刷新

| 属性 | 规范 |
| --- | --- |
| 触发方式 | 在 ScrollView 顶部继续下拉 ≥ 阈值（约 64dp） |
| 刷新范围 | 结余卡片、收支统计、近期账单同时刷新 |
| 刷新指示器 | Flutter `RefreshIndicator`，颜色使用 `$accent-green` |
| 刷新成功 | 数据更新，指示器消失 |
| 刷新失败 | 指示器消失，Toast 提示"刷新失败，请重试" |

---

## 9. 动效规格

| 元素 | 动效类型 | 时长 | 曲线 |
| --- | --- | --- | --- |
| 页面数据加载完成 | 内容区 FadeIn | 300ms | `easeOut` |
| 快速记账弹窗进入 | 底部 SlideUp | 350ms | `easeOutCubic` |
| 快速记账弹窗退出 | 底部 SlideDown | 250ms | `easeInCubic` |
| Tab 切换 | Fade 或 Slide | 200ms | `easeInOut` |
| FAB 按下 | Scale 1.0 → 0.95 | 100ms | `easeIn` |
| FAB 松开 | Scale 0.95 → 1.0 | 150ms | `easeOut` |
| Shimmer 动画 | 线性左→右循环 | 1200ms/loop | `linear` |

## 10. 边界行为汇总

| 场景 | 处理方式 |
| --- | --- |
| 网络断开时进入首页 | 显示错误态 + 展示缓存的上次数据（若有） |
| 金额字段为 null | 显示 "--" |
| 用户名为空 | 只显示问候语，不显示用户名行 |
| 账单分类未知 | 使用 `wallet` 图标 + 灰色底色 |
| 账单条数超过 5 条 | 首页只展示 5 条，其余通过"查看全部"访问 |
| 未登录访问首页 | 重定向至登录页，首页数据不可见 |
| 快速连续点击 FAB | 防抖处理，弹窗只打开一次 |
