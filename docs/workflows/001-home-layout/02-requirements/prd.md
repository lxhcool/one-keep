## 2. 背景与目标

### 业务背景

OneKeep 是一款面向个人用户的移动端记账应用，帮助用户掌握每月收支概况，养成记账习惯。首页是用户每次打开应用的第一落点，承担"数据概览 + 快速发起记账"两大核心职责。

### 用户痛点

- 用户不知道本月结余状况，需要多个步骤才能看到
- 记录一笔账操作路径太长，容易放弃
- 近期账单无法快速回顾，容易遗忘消费记录

### 目标（可衡量）

| 目标 | 衡量标准 |
| --- | --- |
| 用户打开App后 3 秒内能看到本月结余 | 首页结余卡片数据首次渲染 ≤ 3s（P90） |
| 发起记账操作路径不超过 1 步 | FAB 点击 → 记账弹窗，0 中间页面 |
| 近期账单一次最多展示最近 5 条 | 首页账单列表固定展示当日及前一日共 ≤ 5 条 |

### 非目标

- 首页不提供账单筛选、分类统计图表（由"统计"页承担）
- 首页不提供修改/删除账单功能
- 不涉及多账户切换（v1.0 单账户）

## 3. 用户角色与关键场景

### 核心角色

**普通记账用户**：每日有 1–5 次记账需求，希望快速完成并随时查看收支概况。

### 次要角色

**只读浏览用户**：仅查看收支数据，不主动记账。

### 关键场景

| 编号 | 角色 | 行为 | 期望价值 |
| --- | --- | --- | --- |
| S-01 | 普通记账用户 | 早晨打开 App 查看昨日消费 | 快速了解前日支出，无需翻账单细节 |
| S-02 | 普通记账用户 | 购物后立即记一笔支出 | 点 FAB → 填写金额分类 → 保存，30秒内完成 |
| S-03 | 普通记账用户 | 月底查看本月结余 | 结余卡片一眼看出是否超支 |
| S-04 | 只读浏览用户 | 查看本月收入合计 | 收支统计卡片直观显示 |

### 用户故事

- **US-01**：作为用户，我想在打开 App 时立刻看到本月结余和环比变化，以便快速判断财务状况。
- **US-02**：作为用户，我想点击一个按钮就能发起记账，以便降低记账门槛。
- **US-03**：作为用户，我想在首页看到最近几笔账单，以便快速唤起消费记忆。
- **US-04**：作为用户，我想在首页看到本月收入和支出的汇总，以便了解整体收支比。

## 4. 范围

### In Scope

- 用户头像、问候语、用户名展示
- 搜索入口、通知入口（图标展示，行为待明确）
- 本月结余卡片（金额 + 环比变化）
- 本月收入 / 支出统计双卡展示
- 近期账单列表（按日期分组，首页显示最多 5 条）
- "查看全部"跳转至账单页面
- 底部 TabBar 全局导航（首页/统计/账单/我的）
- FAB 快速记账入口（弹出记账弹窗）
- 加载态、空态、错误态

### Out of Scope

- 账单详情页
- 搜索功能具体实现（v1.0 入口占位）
- 通知列表功能（v1.0 入口占位）
- 多账户切换
- 统计图表（由统计页承担）
- 账单编辑/删除（由账单页承担）

## 7. 数据与校验规则

| 字段 | 来源 | 规则 | 默认值 / 错误提示 |
| --- | --- | --- | --- |
| 本月结余 | 后端 API | 数值类型，保留2位小数，支持负值 | 加载中显示骨架；失败显示"--" |
| 结余环比 | 后端 API | 浮点，百分比，可正可负 | 数据缺失时隐藏变化行 |
| 本月收入 | 后端 API | 数值，精确到元（整数） | 同上 |
| 本月支出 | 后端 API | 数值，精确到元（整数），展示时为正值 | 同上 |
| 用户名 | 本地用户 Profile | 字符串，最大 20 字符 | 超长时末尾截断显示 |
| 问候语 | 客户端本地计算 | 06:00–11:59 → 早上好；12:00–17:59 → 下午好；18:00–05:59 → 晚上好 | — |
| 账单列表 | 后端 API | 按日期降序，首页取最近 5 条 | 空列表显示空态 |
| 账单金额 | 后端 API | 收入为正数（绿色），支出为负数（默认色） | — |
| 账单分类图标 | 本地枚举/配置 | 按 category 字段匹配图标和颜色 | 未知分类显示默认图标 |

| --- | --- | --- |
| 后端聚合结余/收支接口响应慢 | 首页加载超时，用户流失 | 骨架屏 + 缓存上次数据兜底；接口超时限制 5s |
