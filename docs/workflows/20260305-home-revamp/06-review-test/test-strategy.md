# 测试策略

## 风险矩阵

| 风险点 | 风险级别 | 触发条件 | 缓解措施 |
| --- | --- | --- | --- |
| 中间“记账”说明位误触导致跳转 | 高 | 点击中间文案区域 | 仅 `+` 绑定点击；文案不绑定事件；加自动化断言 |
| `/transaction/new` 被动态路由误匹配 | 高 | 路由配置顺序错误 | 固定静态路由优先于 `/transaction/:id` |
| 月份切换状态错乱 | 中 | 快速切月导致异步返回乱序 | `token` 机制丢弃过期请求 |
| 空态/错误态不可达 | 中 | 服务返回空/异常时未覆盖 | 组件化状态分支 + 自动化测试 |

## 覆盖优先级

- P0: 记账主入口、路由正确性、月切换核心路径。
- P1: 详情跳转、空态与重试行为。
- P2: 视觉细节与文案一致性。
