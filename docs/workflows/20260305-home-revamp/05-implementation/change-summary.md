# 变更摘要

## 行为变化

- 首页品牌标题统一为 `One Keep`。
- 移除快捷入口宫格（QuickEntryPanel）。
- 底部导航中间“记账”改为说明位：不可点击、不高亮。
- 底部 `+` 按钮成为唯一记账入口。
- 首页数据支持分步加载（先统计后列表）。
- 已落地页面状态机：`loading/success/error/empty`，并提供失败重试与空态引导。
- 流水列表项支持路由跳转到详情页（`/transaction/:id`）。
- `+` 按钮已接入新增流水页（`/transaction/new`），保存后回到首页并刷新数据。

## 兼容性说明

- 对旧接口兼容：保留 `/snapshot` 聚合接口作为降级路径。
- 对导航兼容：四个可点击项路由不变，中间说明位不再触发路由。
- 对数据兼容：金额展示规则统一，不改变底层交易存储语义。

## 已知技术债

- 空态品牌素材暂为占位，需后续替换正式资源。
- 若分步接口尚未落地，前端需暂用聚合接口模拟分步渲染。
- 新增成功后的首页刷新策略需要结合缓存框架进一步优化。
