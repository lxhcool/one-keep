# 实现计划

## 切片计划

1. Slice-01: 首页壳与导航重构
- 目标: 落地底部导航 5 视觉位，其中中间“记账”为说明位，不可点击不高亮。
- 范围: `home` 页面容器、`BottomTabBar`、`CenterAddButton`。
- 产出: UI 结构完成，路由行为与高亮规则生效。

2. Slice-02: 顶部概览与月份切换
- 目标: 展示 `One Keep`、月份、收入/支出；弹层月份选择器可切换。
- 范围: `HomeHeaderSummary`、月份弹层组件、状态管理。
- 产出: 月份切换触发数据刷新，顶部统计同步。

3. Slice-03: 首页数据接口接入（分步返回）
- 目标: 接入 `summary` 与 `dayGroups` 查询，支持先统计后列表。
- 范围: 数据服务层、请求编排、错误重试节流（1秒）。
- 产出: `loading/success/error/empty` 全状态可运行。

4. Slice-04: 日分组流水列表与详情跳转
- 目标: 完成按日期分组渲染与流水项点击详情。
- 范围: `TransactionDaySection`、`TransactionItemRow`、详情页跳转。
- 产出: 列表滚动稳定，金额符号规则正确（收入正、支出负）。

5. Slice-05: 新增流水主流程闭环
- 目标: `+` 按钮进入新增页，提交成功返回首页并刷新。
- 范围: 新增流水页面、`POST /transactions`、返回刷新机制。
- 产出: 记账主流程可端到端使用。

6. Slice-06: 空态/异常态与验收收口
- 目标: 补齐空态、错误态、重试与回归验证。
- 范围: 空态组件、错误提示、自动化断言。
- 产出: 满足阶段2验收标准，可进入阶段6审查测试。

## 依赖关系

- Slice-01 -> Slice-02（页面壳体先稳定）
- Slice-02 -> Slice-03（交互状态先就位，再接接口）
- Slice-03 -> Slice-04（列表数据能力就绪后渲染详情）
- Slice-03 -> Slice-05（新增成功后的刷新依赖首页查询能力）
- Slice-04 + Slice-05 -> Slice-06（收口验证）

## 回滚方案

- UI 回滚: 保留旧版 `home` 路由开关（feature flag: `home_revamp_enabled`）。
- 接口回滚: 查询优先走 `/snapshot` 聚合接口，分步接口异常时降级到聚合接口。
- 交互回滚: 若新增流程不稳定，保留 `+` 按钮但临时跳转旧版新增页。
- 数据安全: 新增流水写接口保持幂等键支持，避免重复提交。
