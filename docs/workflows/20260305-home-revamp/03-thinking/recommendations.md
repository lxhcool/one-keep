# 思考审查 - 建议清单

## 交互优化建议

- 中间“记账”说明位使用禁用态样式，并明确与 `+` 按钮的从属关系（文案/间距/层级）。
- 月份切换时保留旧数据并在局部显示加载态，避免整页闪烁。
- 流水列表首屏建议优先渲染最近日期分组，减少感知等待时间。

## 需求补齐建议

- 明确月份切换器交互形式与返回行为（是否记忆上次选择）。
- 明确流水点击后目标页（仅详情 or 可直接编辑）。
- 明确空状态与错误态的标准文案、图标和按钮文案。

## 优先级建议

- P0: 中间“记账”说明位不可点击不高亮、`+` 主流程可用、月份切换可用。
- P1: 空态/错误态一致性、流水详情跳转、局部加载体验优化。
- P2: 反馈动效细节（禁用点击反馈、过渡动画、骨架屏精细化）。
