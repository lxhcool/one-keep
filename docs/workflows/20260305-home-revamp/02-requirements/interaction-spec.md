# 交互规格

## 交互流程

1. 首页初始进入
- 进入 `home` 页面。
- 默认加载当前月份数据（统计 + 日分组流水）。
- 加载成功后展示顶部概览、列表、底部导航和悬浮 `+`。

2. 月份切换流程
- 用户点击月份区域。
- 打开月份选择器（具体形态待定：下拉/弹层）。
- 用户选择目标月份后关闭选择器。
- 触发数据重载并更新顶部统计与列表。

3. 流水浏览流程
- 用户上下滑动列表查看不同日期分组与流水项。
- 用户点击某条流水，进入流水详情（或编辑）页面。

4. 新增流水流程
- 用户点击底部中心悬浮 `+`。
- 进入新增流水页面。
- 新增成功返回首页后，首页数据按当前月份刷新。

5. 底部导航切换流程
- 用户点击 `明细/图表/发现/我的` 任一项。
- 执行路由切换。
- 导航选中态随当前页面更新。
- 中间“记账”说明位不可点击，不触发跳转。

## 状态变化

- `selectedMonth`: 当前月份。
- `summary`: `{ income, expense }`。
- `dayGroups`: 按日期分组的流水数组。
- `activeBottomTab`: 当前底部导航项（仅 `明细/图表/发现/我的`）。
- `pageStatus`: `loading | success | error | empty`。

状态转换规则：
- 首次进入: `loading -> success|empty|error`。
- 切换月份: 保持当前页，列表区域进入 `loading`，完成后转 `success|empty|error`。
- 新增流水成功返回: 触发刷新，最终回到 `success`。

## 异常处理

1. 数据请求失败
- 展示错误提示（如“加载失败，请重试”）。
- 提供显式 `重试` 按钮；点击后重新请求当前月份数据。

2. 月份切换失败
- 保持旧数据不清空。
- 提示“切换失败”，允许用户再次选择或重试。

3. 空数据
- 当 `dayGroups` 为空时展示空状态组件。
- 空状态包含引导动作：点击“去记一笔”触发新增流水流程。

4. 导航/跳转失败（极端情况）
- 保持当前页可操作。
- 记录错误日志并提示用户稍后重试。
