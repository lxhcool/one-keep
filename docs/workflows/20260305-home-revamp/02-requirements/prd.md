# PRD

## 目标

- 完成 `home` 页面改版需求定义，统一信息层级与核心操作路径。
- 首页聚焦两件事：查看当月收支概览、快速进入新增流水。
- 与阶段 1 组件边界保持一致，确保后续架构与实现可直接落地。

## 功能需求

1. 顶部概览区

- 展示品牌标题：`One Keep`。
- 展示当前选择月份（如 `2025年 07月`），支持触发月份切换。
- 展示当月收入、支出金额两项指标。

2. 流水列表区（按日分组）

- 按日期分组展示流水（例如 `07月26日 星期六`）。
- 每个分组展示当日支出汇总金额。
- 每条流水展示分类图标、分类名称、金额。
- 列表支持纵向滚动浏览。

3. 底部导航区

- 提供 5 个视觉位：`明细/图表/记账（说明位）/发现/我的`。
- 仅 `明细/图表/发现/我的` 可点击并可高亮。
- 中间 `记账` 视觉位不可点击、不高亮，仅用于说明中间 `+` 是记账入口。

4. 主操作按钮

- 底部中间固定悬浮 `+` 按钮。
- 点击后进入新增流水流程。

5. 页面状态

- 支持 `加载中`、`加载成功`、`加载失败` 三态。
- 加载失败时提供可见的重试入口。
- 当某月无流水时展示空状态文案与引导动作（去记一笔）。

## 验收标准

1. 信息展示

- 进入首页后可看到 `One Keep` 标题、月份、收入、支出。
- 收入和支出金额与所选月份数据一致。

2. 月份切换

- 点击月份可切换至其他月份。
- 切换后顶部统计与流水列表同步刷新，且数据归属月份一致。

3. 列表展示

- 流水按日期分组渲染，分组头含日期与当日支出。
- 流水项字段完整（图标、名称、金额），金额正负格式正确。

4. 主操作与导航

- 底部悬浮 `+` 按钮在首页始终可见并可点击。
- 点击 `+` 后进入新增流水流程。
- 底部导航仅四个可点击项可正常跳转，当前页高亮正确。
- 中间“记账”说明位点击无跳转（或无点击态反馈）。

5. 异常与空态

- 数据加载失败可见错误提示与重试按钮。
- 无流水数据时显示空态，不出现布局错位或白屏。
