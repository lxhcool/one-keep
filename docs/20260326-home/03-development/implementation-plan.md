# 开发实施计划

## 目标

- 用最小可控范围落地 home、bills、stats 三页主路径。
- 固定顺序：先服务端，再客户端，最后联调。

## 阶段拆分

| 阶段 | 目标 | 主要产出 |
| --- | --- | --- |
| P1 | 服务端聚合与查询能力就绪 | /api/home/summary, /api/bills, /api/stats/overview 契约与实现 |
| P2 | 客户端主壳层与首页就绪 | MainScaffold, HomePage, 共享交易模型 |
| P3 | 账单页与统计页接入 | BillsPage, StatsPage, 筛选与图表组件 |
| P4 | 状态补齐与联调 | 骨架、空态、错误态、自检记录 |

## 需求映射

| 需求 | 服务端动作 | 客户端动作 |
| --- | --- | --- |
| FR-001 / CR-001 | 提供首页 summary | 渲染首页卡片和最近交易 |
| FR-006 / CR-009 | 返回筛选账单列表 | 切换筛选并刷新列表 |
| FR-004 / CR-006 | 返回趋势数据 | 切换收入/支出图表 |
| FR-010 / AC-008 | 区分空结果和错误 | 展示空态和错误态 |

## 交付顺序建议

1. 首页主链路。
2. 账单明细浏览。
3. 统计洞察。
4. FAB 入口预留。
