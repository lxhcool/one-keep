# 项目仓库结构说明

本项目按照通用研发仓库规范管理，根目录包含以下核心目录：

```text
.
├── source/       # 源代码规范入口与目录映射说明
├── docs/         # 项目文档
├── assets/       # 数字原始资产归档入口
├── client/       # Flutter 客户端工程
├── server/       # Node.js/Fastify 服务端工程
├── scripts/      # 构建、部署、文档生成脚本
├── README.md     # 项目介绍、功能、部署和使用说明
└── package.json  # 根项目命令入口
```

## 目录职责

- `source/`：源代码规范入口。当前以说明文件映射现有源码位置，避免破坏已配置的构建、测试和部署命令。
- `docs/`：项目说明、结构规范、需求规划、部署记录和研发文档。
- `assets/`：设计文件、原始素材、图标、截图等数字原始资产的归档入口。
- `client/`：Flutter 客户端，包括 `lib/`、`assets/`、`test/` 和各平台工程。
- `server/`：服务端 API，包括 `src/`、`prisma/` 和服务端配置。
- `scripts/`：本地部署、生成材料等自动化脚本。

## 版本管理要求

- 依赖目录、构建产物、缓存文件、临时输出和本地敏感配置必须通过 `.gitignore` 过滤。
- Commit 日志应准确描述修改内容，建议使用 `feat:`、`fix:`、`docs:`、`refactor:` 等前缀。
- 不应提交本地环境变量、密钥、临时截图、软著提交包、构建产物等非源码内容。
