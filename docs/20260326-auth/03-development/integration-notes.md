## 问题与阻塞

| 问题 | 影响 | 当前处理 | 下一步 |
| --- | --- | --- | --- |
| 注册成功后去向未确认 | 影响 register 成功态和联调断言 | 文档中保留两种策略 | 开发前由产品确认 |
| 密码规则未冻结 | 影响字段级校验一致性 | 先按“最小长度 + 合法字符”抽象 | 服务端给出最终规则 |
| 微信登录未确认首发 | 影响第三方 SDK 和联调排期 | 默认不阻塞账号密码主链路 | 首发后置或按开关控制 |
| 当前无代码改动 | 无法执行真实联调 | 已补齐联调项清单 | 进入编码后执行 |

## 推荐联调顺序

1. 注册：先验证字段校验、冲突错误、成功返回。
2. 登录：再验证用户名登录和手机号登录两条路径。
3. 路由：验证登录注册互跳与已登录重定向。
4. 微信登录：若进入范围，再单独联调授权链路。

## 联调最小检查清单

| 检查项 | 预期 |
| --- | --- |
| 用户名登录 | 合法用户名 + 密码可进入主应用 |
| 手机号登录 | 合法手机号 + 密码可进入主应用 |
| 注册字段冲突 | 用户名或手机号冲突时给出明确提示 |
| 表单错误 | 字段错误就地提示，不只靠 toast |
| 认证页互跳 | 去注册/去登录路由稳定 |
| 微信入口开关 | 关闭时隐藏或禁用，开启时可发起授权 |
