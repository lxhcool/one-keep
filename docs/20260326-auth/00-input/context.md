# 输入上下文

## 业务目标

- 为 OneKeep 建立独立的认证功能文档，覆盖登录、注册和可选微信登录接入。
- 让未登录用户可以完成账户创建并进入记账主流程，让已注册用户可通过用户名或手机号登录。
- 将认证能力从 home/stats/bills 主路径需求中拆分，便于独立排期与实现。

## 用户对象

- 首次使用 OneKeep 的新用户。
- 已注册且需要再次登录的老用户。
- 使用手机号注册、使用用户名或手机号登录的个人用户。
- 后续可能通过微信快捷登录的移动端用户。

## 约束条件

- 当前没有认证页面设计稿，所有页面结构和交互仅能基于需求推导。
- 用户明确要求本轮只补文档，不开发代码。
- 注册字段固定为手机号、用户名、昵称、密码。
- 登录字段固定为用户名或者手机号、密码。
- 微信登录当前仅作为可选方案记录，不默认视为首发必做项。

## 原始材料

- 用户补充需求：
	- 注册：手机号，用户名，昵称，密码。
	- 登录：用户名或者手机号，密码。
	- 可考虑接入微信登录。
- 现有项目上下文：OneKeep 主应用已有 home/stats/bills 三个主页面需求文档，但无认证设计稿与实现。
