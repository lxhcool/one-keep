# 交互规格

## 1. 主流程

### 1.1 登录主流程

1. 用户进入登录页。
2. 页面展示品牌头、账号输入框、密码输入框和登录按钮。
3. 用户输入用户名或手机号，以及密码。
4. 点击登录按钮后进入提交中状态。
5. 登录成功后持久化登录态并进入主应用首页。
6. 用户可通过底部入口切换到注册页。

### 1.2 注册主流程

1. 用户进入注册页。
2. 页面展示手机号、用户名、昵称、密码四个输入项和注册按钮。
3. 用户填写字段并触发实时校验或提交校验。
4. 点击注册按钮后进入提交中状态。
5. 注册成功后，进入既定后续流程：跳登录页或直接进入主应用。
6. 用户可通过底部入口切换到登录页。

### 1.3 微信登录主流程

1. 用户在登录页点击微信登录按钮。
2. 客户端拉起微信授权流程。
3. 授权成功后，将授权结果提交服务端换取应用登录态。
4. 登录成功后进入主应用；若为新用户，可进入补充资料流程或直接完成账号创建。

## 2. 分支与异常流程

### 2.1 登录页

| 场景 | 触发条件 | 预期行为 |
| --- | --- | --- |
| 账号为空 | 点击登录时 account 为空 | 阻止提交并展示字段错误 |
| 密码为空 | 点击登录时 password 为空 | 阻止提交并展示字段错误 |
| 账号格式不合法 | 输入明显不满足手机号或用户名规则 | 展示字段级错误，允许继续编辑 |
| 认证失败 | 服务端返回账号或密码错误 | 展示表单级错误，不清空全部输入 |
| 网络异常 | 登录请求失败 | 展示网络错误与重试提示 |
| 微信未开放 | 页面配置关闭微信登录 | 不展示微信按钮或展示禁用态 |

### 2.2 注册页

| 场景 | 触发条件 | 预期行为 |
| --- | --- | --- |
| 手机号格式错误 | 输入手机号不合法 | 展示字段级错误 |
| 用户名冲突 | 服务端校验失败 | 展示用户名已占用提示 |
| 手机号冲突 | 服务端校验失败 | 展示手机号已注册提示 |
| 昵称为空 | 提交时昵称为空 | 阻止提交并提示 |
| 密码不合规 | 密码不满足规则 | 展示密码规则提示 |
| 注册成功后策略 | 接口成功 | 跳登录页或自动登录，遵循产品决策 |

### 2.3 通用流程

| 场景 | 触发条件 | 预期行为 |
| --- | --- | --- |
| 重复提交 | 主按钮处于 loading 状态时再次点击 | 不重复发起请求 |
| 页面切换 | 点击“去登录”或“去注册” | 进行路由切换并保留必要参数 |
| 已登录访问认证页 | 用户已有有效登录态 | 重定向到主应用，避免重复进入认证页 |

## 3. 状态变化

| 页面/组件 | 默认 | 加载 | 空 | 错误 | 选中/激活 | 禁用 |
| --- | --- | --- | --- | --- | --- | --- |
| 账号输入框 | 占位文案 | 不适用 | 空值默认态 | 字段错误态 | 聚焦态 | 提交中可选禁用 |
| 手机号输入框 | 占位文案 | 不适用 | 空值默认态 | 字段错误态 | 聚焦态 | 提交中可选禁用 |
| 用户名输入框 | 占位文案 | 可选校验中 | 空值默认态 | 字段错误态 | 聚焦态 | 提交中可选禁用 |
| 昵称输入框 | 占位文案 | 不适用 | 空值默认态 | 字段错误态 | 聚焦态 | 提交中可选禁用 |
| 密码输入框 | 密文默认态 | 不适用 | 空值默认态 | 字段错误态 | 显示/隐藏切换 | 提交中可选禁用 |
| 主按钮 | 可点击默认态 | loading | 不适用 | 不适用 | 按下态 | 表单非法时禁用 |
| 微信登录按钮 | 可点击默认态 | loading | 不适用 | 授权失败提示 | 按下态 | 未开放时隐藏或禁用 |
| 页底切换入口 | 默认态 | 不适用 | 不适用 | 不适用 | 按下态 | 不适用 |

## 4. 数据映射与展示规则

| 服务端字段 | 客户端展示位置 | 展示规则 | 备注 |
| --- | --- | --- | --- |
| account | 登录页账号输入框 | 可输入用户名或手机号 | 前端不要求单独选择类型 |
| phone | 注册页手机号输入框 | 仅接受手机号格式 | 可使用数字键盘 |
| username | 注册页用户名输入框 | 展示唯一性冲突提示 | 规则需与服务端一致 |
| nickname | 注册页昵称输入框 | 作为展示名输入 | 一般允许重复 |
| password | 登录/注册密码框 | 默认密文显示 | 支持显隐切换 |
| accessToken | 登录成功后本地存储 | 不在 UI 直接展示 | 存储策略待开发阶段细化 |
| errorCode / message | 表单级或字段级错误 | 精准映射错误文案 | 不使用模糊失败提示替代 |

## 5. 页面跳转建议

| 来源 | 触发动作 | 目标 | 当前结论 |
| --- | --- | --- | --- |
| login | 点击去注册 | register | 可确认 |
| register | 点击去登录 | login | 可确认 |
| login | 登录成功 | 主应用首页 | 可确认 |
| register | 注册成功 | 登录页或主应用首页 | 待确认 |
| login | 点击微信登录 | 微信授权流程 | 可选能力 |
| login | 点击忘记密码 | 找回密码页 | 待确认 |

## 6. 可直接用于开发的默认约束

- 登录页账号输入框统一承接用户名和手机号，不做双入口拆分。
- 登录和注册主按钮在提交期间必须禁用，防止重复提交。
- 字段错误优先做就地提示，接口错误再补充 toast 或表单级提示。
- 登录/注册页应共享认证基础组件和主题 token。
- 微信登录默认作为可配置入口处理，不把 UI 结构绑死到首发版本。

## 7. 待确认交互

- 注册成功后是否自动登录。
- 是否需要忘记密码入口。
- 是否需要用户协议勾选。
- 用户名是否允许纯数字、下划线、中文等字符集。
- 微信登录新用户是否要补充手机号或昵称。
