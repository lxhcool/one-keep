# 需求文档

## 1. 文档信息

- 功能名称：OneKeep 登录/注册认证流程
- 版本：v0.1
- 负责人：GitHub Copilot
- 日期：2026-03-26
- 关联页面：login, register

## 2. 背景与目标

- 业务背景：主应用已有首页、统计页、账单页主路径需求，但缺少独立认证流程定义，导致用户无法完成首次进入与身份恢复。
- 用户痛点：新用户缺少注册入口与字段规则，老用户缺少统一登录入口，第三方快捷登录策略也未定义。
- 目标：
	- 建立手机号 + 用户名体系下的注册流程。
	- 建立用户名或手机号 + 密码的登录流程。
	- 将微信登录作为可选增强能力记录，支持独立排期。
- 非目标：
	- 本轮不展开找回密码、短信验证码登录、风控验证、人脸识别等扩展认证能力。
	- 本轮不进入代码实现。

## 3. 用户角色与场景

- 新用户：首次打开应用，需要完成注册并进入主应用。
- 老用户：已拥有账号，通过用户名或手机号登录。
- 移动端快捷登录用户：期望通过微信授权快速进入应用。

关键场景：

- 用户输入手机号、用户名、昵称和密码完成注册。
- 用户使用用户名或手机号 + 密码完成登录。
- 用户点击微信登录发起第三方授权。
- 用户在登录页和注册页之间切换。

## 4. 范围

### In Scope

- 登录页与注册页的信息结构、组件拆分和页面交互定义。
- 认证服务端接口、字段校验和错误返回约束。
- 微信登录作为可选能力的需求占位。

### Out of Scope

- 找回密码、短信验证码登录、邮箱注册、注销账号。
- 风控验证码、设备管理、登录历史、安全中心。
- 代码实现与第三方 SDK 集成细节。

## 5. 服务端需求

### 5.1 服务端职责

- 提供账号密码登录接口，支持用户名或手机号识别。
- 提供注册接口，接收手机号、用户名、昵称、密码并完成账号创建。
- 提供用户名、手机号唯一性和格式校验规则。
- 提供明确的认证失败、账号已存在、参数不合法等错误码。
- 若启用微信登录，提供授权换取应用登录态的服务端能力。

### 5.2 功能需求

| ID | 模块 | 需求描述 | 优先级 | 备注 |
| --- | --- | --- | --- | --- |
| FR-001 | 登录 | 服务端需提供登录接口，支持账号字段传入用户名或手机号，并结合密码进行认证。 | P0 | 认证核心流程 |
| FR-002 | 登录态返回 | 登录成功后需返回用户基础信息、访问令牌和必要的会话信息。 | P0 | 供客户端进入主应用 |
| FR-003 | 注册 | 服务端需提供注册接口，接收手机号、用户名、昵称、密码并创建用户。 | P0 | 注册核心流程 |
| FR-004 | 唯一性校验 | 服务端需校验手机号和用户名是否已被占用，并返回明确错误。 | P0 | 防止重复注册 |
| FR-005 | 参数校验 | 服务端需校验手机号格式、用户名格式、昵称长度、密码规则。 | P0 | 校验规则需统一 |
| FR-006 | 错误返回 | 服务端需区分参数非法、认证失败、账号冲突、未授权和系统异常。 | P0 | 便于客户端精确提示 |
| FR-007 | 微信登录 | 若启用微信登录，服务端需支持微信授权结果换取 OneKeep 登录态。 | P1 | 预留能力 |

### 5.3 接口与数据契约

| 接口 | 方法 | 输入 | 输出 | 校验/权限 | 异常说明 |
| --- | --- | --- | --- | --- | --- |
| /api/auth/login | POST | account, password | user, accessToken, refreshToken? | account 和 password 必填 | 400 参数非法；401 认证失败；500 服务异常 |
| /api/auth/register | POST | phone, username, nickname, password | userId, user, accessToken? | phone/username 唯一，字段规则合法 | 400 参数非法；409 用户名或手机号冲突；500 服务异常 |
| /api/auth/wechat | POST | authCode | user, accessToken, isNewUser | 需校验微信授权码 | 400 参数非法；401 授权失败；500 服务异常 |
| /api/auth/check-availability | GET/POST | phone?, username? | phoneAvailable, usernameAvailable | 至少一个字段必填 | 400 参数非法；500 服务异常 |

### 5.4 数据与规则

| 字段/对象 | 校验规则 | 业务规则 | 备注 |
| --- | --- | --- | --- |
| phone | 合法手机号格式，必填 | 注册唯一；登录时可作为 account | 具体国家区号范围待确认 |
| username | 非空，长度与字符集合法 | 注册唯一；登录时可作为 account | 是否允许纯数字待确认 |
| nickname | 非空，长度合法 | 注册成功后作为默认展示名 | 是否允许重复通常可接受 |
| password | 非空，满足最小长度与复杂度规则 | 仅服务端校验，不回传明文 | 复杂度标准待确认 |
| accessToken | 非空字符串 | 登录或注册成功后返回 | 是否同时返回 refreshToken 待确认 |
| isNewUser | 布尔值 | 微信登录新用户标识 | 仅微信登录返回 |

### 5.5 服务端业务规则与实现顺序

| ID | 规则/依赖 | 要求 | 优先级 | 备注 |
| --- | --- | --- | --- | --- |
| SR-001 | account 识别 | 登录接口需自动识别 account 是手机号还是用户名，不要求客户端额外传类型。 | P0 | 降低客户端复杂度 |
| SR-002 | 冲突错误 | 用户名冲突与手机号冲突应可区分，便于前端给出明确提示。 | P0 | 注册体验关键 |
| SR-003 | 密码规则 | 密码规则需由服务端统一定义并下发给客户端实现同构校验。 | P0 | 避免前后端不一致 |
| SR-004 | 登录态签发 | 注册成功后是否直接签发登录态必须统一，不允许不同端策略不一致。 | P1 | 需产品确认 |
| SR-005 | 开发顺序 | 优先完成注册与登录，再决定是否接入微信登录。 | P1 | 控制首发复杂度 |

## 6. 客户端需求

### 6.1 页面与组件职责

- 登录页负责接收账号与密码，完成登录提交、错误提示和注册跳转。
- 注册页负责接收手机号、用户名、昵称和密码，完成注册提交与登录跳转。
- 登录/注册页面共享认证视觉头部、输入框、密码框、主按钮与底部切换组件。
- 微信登录入口默认只在登录页出现，是否显示由配置或产品决策控制。

### 6.2 交互需求

| ID | 触发条件 | 主流程 | 分支/异常流程 | 界面反馈 |
| --- | --- | --- | --- | --- |
| CR-001 | 用户进入登录页 | 展示账号输入、密码输入和登录按钮 | 若已登录则不进入此页 | 默认态展示 |
| CR-002 | 用户点击登录按钮 | 提交 account + password | 字段非法或认证失败时阻止进入主应用 | 按钮 loading、错误提示 |
| CR-003 | 用户点击“去注册” | 跳转注册页 | 路由不可达时提示错误 | 页面跳转 |
| CR-004 | 用户进入注册页 | 展示手机号、用户名、昵称、密码四字段和注册按钮 | 若已登录则不进入此页 | 默认态展示 |
| CR-005 | 用户点击注册按钮 | 提交注册请求 | 字段非法、账号冲突时展示错误 | 按钮 loading、字段或表单错误 |
| CR-006 | 用户点击“去登录” | 跳转登录页 | 路由不可达时提示错误 | 页面跳转 |
| CR-007 | 用户修改字段 | 实时或提交时校验字段合法性 | 校验失败则保留在当前页 | 字段级错误提示 |
| CR-008 | 用户点击微信登录 | 发起微信授权 | 若未开放则提示暂未开放 | 按钮 loading 或提示 |

### 6.3 状态与接口对接

| 场景 | 依赖接口 | 成功态 | 异常态 | 重试/兜底 |
| --- | --- | --- | --- | --- |
| 登录提交 | /api/auth/login | 进入主应用并持久化登录态 | 表单错误/账号密码错误 | 用户可修改后再次提交 |
| 注册提交 | /api/auth/register | 进入登录页或直接进入主应用 | 字段冲突/规则错误 | 用户修正后再次提交 |
| 微信登录 | /api/auth/wechat | 进入主应用或进入补充资料流程 | 授权失败/服务异常 | 支持重试或回退账号登录 |
| 可用性校验 | /api/auth/check-availability | 字段可用状态更新 | 网络失败时不阻塞输入 | 以最终提交校验为准 |

### 6.4 客户端业务规则

| ID | 规则 | 说明 | 优先级 |
| --- | --- | --- | --- |
| CRR-001 | 登录账号共用输入框 | 用户名和手机号必须共用同一个登录账号输入框。 | P0 |
| CRR-002 | 字段错误就地提示 | 字段规则错误优先在字段下方展示，不只依赖 toast。 | P0 |
| CRR-003 | 提交防重 | 登录和注册提交期间禁止重复点击主按钮。 | P0 |
| CRR-004 | 认证页共用组件 | 登录和注册页共享基础认证组件与主题 token。 | P1 |
| CRR-005 | 微信入口可配置 | 微信登录入口是否显示应可配置，而不是写死。 | P1 |

## 7. 非功能需求

| ID | 类别 | 需求描述 |
| --- | --- | --- |
| NFR-001 | 性能 | 登录、注册首屏需快速可交互，避免复杂动画阻塞输入。 |
| NFR-002 | 安全 | 密码输入必须默认密文显示，不得记录明文日志。 |
| NFR-003 | 可用性 | 错误提示需明确指出是账号错误、密码错误还是字段冲突。 |
| NFR-004 | 可维护性 | 登录和注册共用基础组件，避免重复实现。 |
| NFR-005 | 扩展性 | 第三方登录入口预留 provider 扩展能力，便于后续接入更多平台。 |

## 8. 验收标准

| ID | Given | When | Then |
| --- | --- | --- | --- |
| AC-001 | 用户位于登录页 | 输入合法用户名和密码并点击登录 | 登录成功并进入主应用 |
| AC-002 | 用户位于登录页 | 输入合法手机号和密码并点击登录 | 登录成功并进入主应用 |
| AC-003 | 用户位于登录页 | 账号或密码为空就提交 | 页面阻止提交并展示错误提示 |
| AC-004 | 用户位于注册页 | 输入合法手机号、用户名、昵称、密码并点击注册 | 注册成功，进入下一步既定流程 |
| AC-005 | 用户位于注册页 | 用户名已被占用时提交注册 | 页面提示用户名冲突，不创建账号 |
| AC-006 | 用户位于注册页 | 手机号已被占用时提交注册 | 页面提示手机号冲突，不创建账号 |
| AC-007 | 用户位于登录页 | 点击“去注册” | 成功跳转注册页 |
| AC-008 | 用户位于注册页 | 点击“去登录” | 成功跳转登录页 |
| AC-009 | 微信登录已开启 | 用户点击微信登录并授权成功 | 成功换取 OneKeep 登录态 |
| AC-010 | 微信登录未开启 | 用户访问登录页 | 页面不展示微信入口或展示明确的未开放状态 |
