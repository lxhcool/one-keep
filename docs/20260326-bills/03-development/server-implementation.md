# 服务端实现

## 本轮目标

- 基于账单页需求形成独立服务端实施方案。
- 明确 /api/bills 的筛选、搜索、分组和排序规则。

## 接口与数据改动

| 需求编号 | 接口/模块 | 改动内容 | 当前状态 | 备注 |
| --- | --- | --- | --- | --- |
| FR-001, FR-002, FR-003, FR-004, FR-005 | /api/bills | 新增账单列表接口，支持 filterType、query、month、cursor | 方案已定义 | 账单页唯一核心接口 |

## 推荐模块拆分

| 模块 | 职责 | 备注 |
| --- | --- | --- |
| bills-query-service | 查询账单列表并做分组 | 核心列表服务 |
| search-service | 处理关键字检索 | 默认叠加当前筛选 |
| grouping-service | 生成 groupedBills 和 summary | 口径需稳定 |
