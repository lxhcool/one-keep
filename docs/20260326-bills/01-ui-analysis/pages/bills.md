## 9. 响应式与可访问性说明

- 断点假设：当前仅覆盖手机单列账单流，不包含多列或平板适配。
- 键盘与焦点预期：搜索按钮、筛选项、账单项、底部导航、FAB 均应可聚焦。
- 可读性与对比度风险：深色页日期汇总和次级元信息对比偏低；浅色页整体安全，但筛选 chip 文案较小。
- 可访问性建议：收入/支出不能仅依赖颜色区分，需保留正负号和语义描述；账单项应让屏幕阅读器能朗读“名称 + 分类 + 时间 + 金额 + 收支方向”。

## 11. 缺失项说明

以下内容无法仅凭当前设计稿完整判断，需要额外补充：

1. 搜索展开态、搜索结果页和无结果页。
2. 支出/收入筛选选中态的完整页面效果。
3. 列表加载、空态、错误态和刷新逻辑。
4. 账单详情、编辑、删除、撤销等后续操作。
5. 日期分组是否可折叠，以及长列表滚动时的吸顶策略。
6. 搜索与筛选是否支持组合使用及其优先级。
